# Contributing to Car Rental Management System

First off, thank you for considering contributing to the Car Rental Management System! It's people like you who make the open-source community such an amazing place to learn, inspire, and create.

To maintain a healthy and productive development environment, please review the following guidelines before submitting a contribution.

---

## 🛠️ How Can I Contribute?

### 🐛 Reporting Bugs
If you find a bug or unexpected behavior:
1. Search the **Issues** tab to make sure it hasn't already been reported.
2. If it is new, open a new issue.
3. Use a clear, descriptive title.
4. Describe the steps required to reproduce the bug, including your local PHP/MySQL environment version.
5. Provide screenshots or error messages if applicable.

### 💡 Suggesting Enhancements
We are always looking for ways to make the system better!
1. Open an issue with the tag `enhancement`.
2. Explain the feature you would like to see, why it is useful, and how it should work.

### 💻 Pull Requests
If you want to write code and fix issues:
1. **Fork** the repository to your own account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/Car-Rental-Syatem-PHP-MYSQL.git
   ```
3. Create a descriptive branch for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   # OR
   git checkout -b bugfix/your-bug-name
   ```
4. Commit your changes with clear messages:
   ```bash
   git commit -m "Add feature to search cars by price range"
   ```
5. Push your branch to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. Open a **Pull Request (PR)** against the `main` branch of the original repository.

---

## 🎨 Code Style and Standards
* Keep code clean, readable, and well-commented.
* Ensure SQL queries are sanitized to prevent SQL injection vulnerabilities.
* Test your changes locally to ensure database queries run correctly.
* Follow standard PHP formatting guidelines.
